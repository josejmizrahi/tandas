## 2. Duplicated models

**None of substance.** RuulFeatures has no duplicated domain types. All struct names that match (CapabilitySection, ResourceDetailContext, EventDetailPresenter, ResourceInfoRow, RuleEditRouteContext) are *view/coordinator* types that consume `RuulCore`. The doctrine that "Resource link kinds should live in a single registry" is satisfied: `LinkKind` lives only in `PlatformModels/ResourceLink.swift` with a single `.uses` case.

Note: `CapabilityRuleOption` was deliberately renamed from `RuleTemplate` on 2026-05-17 to eliminate a name collision with the universal `RuleTemplate` (`PlatformModels/RuleTemplate.swift`) — collision resolved.

## 4. Hardcoded capability id strings outside `CapabilityCatalog`

These are *legit referrers* (consumers must name caps to gate), but should reference typed constants:

- `PlatformModules/V1Modules.swift:22-109` — 12 string literals (`"rsvp"`, `"check_in"`, `"rotation"`, `"ledger"`, `"consequence"`, `"appeal"`, `"voting"`, `"attendance"`, `"deadline"`, `"assignment"`, `"rules"`, `"basic_fines"`).
- `Capabilities/EventResourceBuilder.swift:24`, `SpaceResourceBuilder.swift:55`, `FundResourceBuilder.swift:43`, `AssetResourceBuilder.swift:27`, `RightResourceBuilder.swift:89` — builder default cap sets.
- `Capabilities/CapabilityResolver+Expanded.swift:174` (`"voting"`), `+PrimaryAction.swift:158` (`"rsvp"`), `+SecondaryActions.swift:228,247`.
- `Resources/Variants/*Variants.swift` — every variant's `attachedCapabilities: Set<String>` is hand-typed (18 catalogs, ~80 hardcoded ids total).
- `Resources/Intents/DefaultIntents.swift` — `requiredCapabilities: ["rsvp"]`, `["check_in"]`, `["ledger","money"]`, `["booking"]`, `["access"]`, `["custody"]`, `["valuation"]`.
- `Repositories/RuleTemplateRepository.swift:215-548` — 12 templates with hardcoded `requiredCapabilities: [...]`.
- `Repositories/RuleRepository.swift:157` — `guard moduleSlug == "basic_fines"`.
- `RuulFeatures/.../UniversalResourceDetailView.swift:545-586` — **~30 literal capability ids** in a single static list for rendering order.
- `RuulFeatures/.../Detail/Sections/*.swift` — every section view uses `caps.contains("rsvp" | "check_in" | "rules" | "rotation" | "location" | "description" | "host_actions" | "money" | "ledger" | "schedule" | "recurrence" | "custody" | "transfer" | "valuation" | "maintenance" | "booking" | "capacity" | "voting")`.
- `RuulFeatures/.../Home/HomeView.swift:421-427` — `modules.contains("rsvp"|"check_in"|"basic_fines"|"slot_assignment"|"common_fund")`.
- `RuulUI/Primitives/RuulInlineActionBar.swift:116`, `RuulSubTabBar.swift:61-64` — hardcoded `id: "rsvp"`/`"rotation"`/`"rules"`/`"history"`.

**Verdict:** the catalog *exists* and centralizes structure, but **no typed constants exist** — every consumer uses raw strings. Catalog should expose `static let id = "rsvp"` on each block (or a `CapabilityID` namespace).

## 5. Hardcoded link kinds outside the registry

`LinkKind` enum is the canonical registry (`PlatformModels/ResourceLink.swift`). The only string literal `"uses"` appears at `Repositories/ResourceLinkRepository.swift:127` — should be `LinkKind.uses.rawValue`. **Single low-severity hit.**

## 6. Untyped metadata abuse (`[String: Any]`)

Inside the audited surface — **zero** occurrences. `[String: Any]` only appears in:
- `RuulCore/OnboardingCompletion.swift:59,74`
- `RuulCore/Repositories/ProfileRepository.swift:62`

Both are Keychain attrs / `from(...).update(...)` payload dicts — legitimate boundary uses. Domain envelopes correctly use the typed `JSONConfig` enum throughout.

**However** there is a related smell: `CapabilityRuleOption.defaultConfig: [String: String]` (`Capabilities/CapabilityBlock.swift:304`) — stringly-typed envelope where `JSONConfig` would be more honest.

## 7. CapabilityCatalog (1,321 LOC) split proposal

The file declares 36 `CapabilityBlock` structs in a single arena. Recommended split:

```
Capabilities/Catalog/
├── CapabilityCatalog.swift        (~100 LOC — wrapper + .v1 array only)
├── Blocks/Event/                  RSVP, CheckIn, Schedule, Recurrence,
│                                  Attendance, Deadline, Reminder, Description,
│                                  HostActions, Location, Cancellation
├── Blocks/Money/                  Money, Ledger, Consequence
├── Blocks/Governance/             Rules, Voting, Approval, Appeal
├── Blocks/Rotation/               Rotation, Assignment, Participants, Swap
├── Blocks/Asset/                  Custody, Maintenance, Valuation,
│                                  Transfer, Access, Delegation, Inventory
├── Blocks/Space/                  Capacity, GuestAccess, Booking, Expiration
└── Blocks/Status/                 Status, History
```

Move `SimpleCapability` (line 109) to `Blocks/_Support.swift`. Each file becomes 100-200 LOC.

## 8. Enum duplication

- `CapabilityAction.Surface` (`CapabilityBlock.swift:226`) has `case moneyTab`; `Destination` (`ResourceIntent.swift:119`) also has `case moneyTab`. Two parallel "where in the app" enums.
- `Destination.RuleCategoryFilter` (`ResourceIntent.swift:127`) defines `priority/approval/access/money/obligation` — overlaps with universal rule template categories in `RuleTemplate.swift` (`UniversalCategory`).
- Rule scope strings: `"resource" | "series" | "group" | "occurrence" | "global_default" | "resource_type"` are stringly-typed in `RuleShape.validScopes`, `GroupPolicy.targetScope`, `RuleTemplate` JSON envelope (~25 raw-string call sites). A `RuleScope` enum exists nowhere — should.

## 9. Codegen folder status

`PlatformModels/Generated/` is **legitimately generated**. All six files begin with `// AUTOGENERATED. Do not edit. Source: ... Run: make gen`. Source-of-truth files mark themselves with `// @codegen:enum`. Generator at `scripts/codegen/emit-swift.ts`. Enforced via Lefthook. Healthy.

What's missing: only 6 enums participate. `ConditionType+Extensions.swift`, `SystemEventType+Extensions.swift`, `ConsequenceType+Extensions.swift` are **hand-edited adjuncts** — soft asymmetry but intentional (display copy / classification logic that the TS schema doesn't know about).

## 11. Beta blockers

None hard-blocking. Soft-blocking-ish:

1. **CapabilityResolver+PrimaryAction / +SecondaryActions**: explicit `switch resourceType` for action dispatch contradicts doctrine — should be capability-driven.
2. **No `CapabilityID` typed namespace**: 100+ raw capability strings across the codebase. One typo silently disables a section.
3. **Rule scope strings untyped**: `"resource"/"series"/"group"/"occurrence"/"global_default"/"resource_type"` appear as raw strings in 25+ places. Easy to add `RuleScope: String` enum.
4. **`CapabilityRuleOption.defaultConfig: [String: String]`** — should be `JSONConfig` for type honesty.

None require schema changes; all four are refactor PRs.
